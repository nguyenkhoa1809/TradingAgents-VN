Hướng dẫn kiến trúc
Multi-Agent Graph

Mỗi quyết định giao dịch đi qua một hội đồng gồm 12 agent chuyên biệt — phân tích, tranh luận, và ra quyết định theo ba giai đoạn tuần tự. Tài liệu này giải thích cách graph hoạt động và cách thêm, bớt, sửa agent.

12 Agent nodes
3 Giai đoạn
2 Vòng debate
1 Shared state
01

Luồng xử lý — ba giai đoạn

Giai đoạn 1 — Phân tích (tuần tự)
Market Analyst → Sentiment Analyst → News Analyst → Fundamentals Analyst
Mỗi analyst dùng tools riêng → ghi kết quả vào market_report / sentiment_report / news_report / fundamentals_report
↓
Giai đoạn 2 — Tranh luận đầu tư
Bull Researcher ⇄ debate Bear Researcher → Research Manager → Trader
Bull ↔ Bear tranh luận max_debate_rounds × 2 lượt → Research Manager tổng hợp → investment_plan
↓
Giai đoạn 3 — Quản lý rủi ro
Aggressive Analyst ⇄ debate Conservative Analyst ⇄ Neutral Analyst → Portfolio Manager
Ba góc nhìn rủi ro tranh luận vòng tròn → Portfolio Manager ra final_trade_decision
02

Shared state — cách agents giao tiếp

Tất cả agents dùng chung một AgentState dict. Mỗi agent đọc state → xử lý → trả về delta dict để cập nhật. File định nghĩa: tradingagents/agents/utils/agent_states.py

Ba loại thay đổi

Loại 1 Sửa instructions của agent có sẵn Độ phức tạp: ⭐

Toàn bộ "hướng dẫn" của mỗi agent nằm trong một chuỗi system_message hoặc prompt bên trong hàm factory. Sửa trực tiếp chuỗi đó — không cần đụng vào graph hay state.

↳ Ví dụ: thêm quy tắc thị trường VN vào Market Analyst. File: tradingagents/agents/analysts/market_analyst.py, dòng 56–84 (biến system_message)
tradingagents/agents/analysts/market_analyst.py python

[THÊM BẤT KỲ INSTRUCTION NÀO VÀO ĐÂY]

- Ưu tiên foreign ownership room (room ngoại) như một catalyst tăng giá
